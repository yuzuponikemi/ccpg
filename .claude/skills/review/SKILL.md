# コードレビュースキル

指定されたファイルまたは変更差分に対して、以下の観点でコードレビューを行ってください:

1. **型ヒント**: すべての関数に型アノテーションが付いているか
2. **docstring**: 日本語で記述されているか、Args/Returns セクションがあるか
3. **テスト**: 対応するテストが存在するか
4. **バグリスク**: 潜在的なバグやエッジケースはないか
5. **命名規約**: 変数名・関数名が適切か（snake_case）

## 出力フォーマット

```
## レビュー結果: <ファイル名>

### 良い点
- ...

### 改善提案
- [ ] ...

### 総合評価
⭐⭐⭐⭐☆ (4/5)
```

引数が指定されていない場合は、`git diff` で変更されたファイルをレビュー対象とします。
